# 充值中心与山海百宝阁内嵌设计

## 目标

- 将用户侧的“钱包”入口、页面标题和相关说明改为“充值中心”。
- 将商城入口命名为“山海百宝阁”。
- 将 `https://pay.ldxp.cn/shop/baixiaosheng` 作为站内商城入口，在 New API 的 `/mall` 路由中内嵌展示。
- 桌面端优先只展示商城白色商品区域，移动端优先保证内容和支付流程完整。

## 范围

### 包含

- 默认前端的用户侧导航文案、充值页面文案和商城页面。
- `/mall` 认证路由、商城 iframe 容器、加载状态和备用外链。
- 桌面端裁剪和移动端完整显示的响应式行为。
- 相关 i18n 文案、路由生成文件和回归测试。

### 不包含

- 不重命后端钱包、余额、充值和订单的数据库字段或 API。
- 不修改管理员支付配置和钱包业务术语。
- 不代理、拆解或重写商城 HTML，不接管商城的登录、购物车、订单和支付逻辑。
- 不修改 `shop.zshai.cc` 当前跳转到原商城地址的服务器配置；内嵌页面直接访问原商城地址，避免额外跳转。

## 导航与文案

- 个人区导航中，`/wallet` 的显示名称从“钱包”改为“充值中心”，路由和图标保持不变。
- `/wallet` 页面用户可见标题和说明使用“充值中心”，余额、充值、支付方式和账单记录的功能保持不变。
- 商城导航和页面标题使用“山海百宝阁”，路由保持 `/mall`。
- 新增文案通过 i18n 键渲染，中文显示指定品牌名，其他语言使用可理解的商城和充值中心翻译。

## 商城页面

### 数据配置

- iframe 入口使用 `https://pay.ldxp.cn/shop/baixiaosheng`。
- 裁剪尺寸和桌面端最小宽度集中存放在商城 feature 的配置文件中，避免分散在组件里。
- 桌面端当前裁剪值为：上 `238px`、左 `335px`、右 `350px`、下 `143px`；可在商城页面更新后按实际边界继续校准。

### 桌面端

- 容器隐藏溢出内容，通过放大 iframe 并向左上平移，只露出白色商品区域。
- 页面保留 New API 的侧边栏和顶部布局，商城内部独立滚动。
- 商品卡片、分类、搜索、下单和支付操作由原商城页面处理。

### 移动端

- 不应用桌面端固定裁剪，iframe 按完整商城页面展示。
- 保证横向内容可见，不裁掉商品选择、下单和支付按钮。
- 商城页面高度至少填满可用视口，避免出现双重短容器。

## iframe 权限与安全

- 使用 sandbox，只开启商城必需的脚本、同源 Cookie、表单、弹窗、下载和用户主动触发的顶层跳转。
- `allow` 仅开启剪贴板写入和支付能力。
- 不向 iframe 注入 New API 的用户信息、Token 或支付密钥。
- 商城支付在原商城域名下完成，New API 仅作为显示容器。

## 加载和失败处理

- iframe 加载前显示简洁的加载状态，加载完成后移除。
- 页面提供“在新窗口打开”备用命令，指向 `https://pay.ldxp.cn/shop/baixiaosheng`。
- 由于跨域 iframe 无法可靠读取内部 DOM，裁剪使用可配置的固定像素值，不实现伪可靠的 DOM 自动定位。
- 若上游修改 `X-Frame-Options` 或 CSP 导致无法内嵌，用户仍可通过备用命令打开商城。

## 测试与验收

- 单元测试校验商城入口、裁剪配置和内部路由配置。
- 执行默认前端 typecheck、相关文件 lint 和生产构建。
- 桌面端验证白色商品区域完整显示，不露出页面顶部导航和店铺头部。
- 移动端验证商品分类、商品列表、下单和支付入口可见且可点击。
- 验证“充值中心”仍可正常发起在线支付，充值记录和余额不受文案变更影响。
- 验证“山海百宝阁”的备用外链能够打开商城。

## 发布与回滚

- 前端构建和服务端嵌入资源与现有 New API 镜像一起发布。
- 发布前备份当前数据库并保留当前运行镜像和容器。
- 上线后验证 `/api/status`、`/wallet`、`/mall` 和商城支付跳转。
- 如新版无法加载商城或影响充值，立即恢复上一个镜像和保留的容器配置。
