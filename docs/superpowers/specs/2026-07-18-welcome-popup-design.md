# 网站迁移欢迎弹窗设计

## 背景

线上旧版本需要向用户通知服务器迁移和新域名启用，但不能把 GitHub 上正在继续开发的版本部署到旧服务器。功能基于 `6623d3b5` 创建的 `production/welcome-popup` 分支实现。

## 目标

- 登录后在统一的已认证布局中显示迁移通知弹窗。
- 告知用户新域名为 `https://zshai.cc`，并引导用户添加微信 `xxz231005`。
- 在弹窗中展示现有客服微信二维码。
- 提供新站链接、关闭和“不再提醒”操作。
- 只改前端，不改数据库、后端 API、代理、佣金、商城和现有使用流程。

## 非目标

- 不把迁移通知接入系统公告管理。
- 不新增用户表字段，也不在服务端保存关闭状态。
- 不强制跳转新站，不影响用户继续使用当前旧站。

## 方案

在 `AuthenticatedLayout` 中挂载一个独立的 `WelcomeMigrationDialog` 组件。组件复用项目现有的 Dialog 基础组件和按钮样式，不复制一套弹窗框架。

组件仅在浏览器环境中读取本地状态：

- `localStorage` 保存用户勾选“不再提醒”的永久关闭标记。
- `sessionStorage` 保存本次浏览器会话已经展示过的标记，避免在路由切换或刷新时反复弹出。
- 如果用户没有永久关闭，新的浏览器会话仍会展示弹窗。

弹窗关闭、点击“我知道了”或点击右上角关闭按钮只结束当前显示；只有勾选“不再提醒”后才写入永久关闭标记。点击“前往新网站”使用新标签页打开 `https://zshai.cc`，不改变当前页面。

## 内容和布局

- 标题：`网站迁移通知`
- 说明：为了给大家带来更稳定、顺畅的使用体验，我们正在更换服务器，并启用了新的域名。新网站目前仍在部署中，完成后请前往新网站重新注册并使用。
- 提醒区：为避免错过新站开放、网站迁移等重要消息，请先添加主播微信。
- 微信号：`xxz231005`
- 二维码：复用 `/customer-service-wechat.jpg`
- 操作：`前往新网站`、`我知道了`
- 选项：`不再提醒`

弹窗使用响应式布局：桌面端显示并排的文字和二维码区域，窄屏下改为纵向排列；二维码保持正方形且不被裁剪。背景使用现有 Dialog 的遮罩和模糊能力。

## 错误处理

- `localStorage` 或 `sessionStorage` 不可用时，弹窗仍可以显示和关闭，但不保证跨刷新记忆状态。
- 新站链接打开失败不影响旧站当前页面。
- 二维码资源加载失败时显示微信号文本，不能让弹窗内容整体消失。

## 验证

- 前端类型检查和现有测试通过。
- 首次进入已认证页面显示弹窗。
- 点击关闭按钮后本次显示结束，路由切换不重复显示。
- 勾选“不再提醒”后刷新和重新打开会话均不再显示。
- 点击新站按钮打开 `https://zshai.cc`，当前旧站页面保持不变。
- 桌面和移动宽度下无内容溢出，二维码和按钮可操作。

## 部署边界

代码只提交到 `production/welcome-popup`。旧服务器部署时只选择这个分支；`integration/upstream-rc21-agent` 和其他开发分支不做合并、不覆盖。
